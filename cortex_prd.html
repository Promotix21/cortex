01 Overview & Goals Core

Cortex solves the context-switching problem that every developer using AI coding assistants faces. When you're managing 5–10 active projects simultaneously — each with their own terminal sessions, AI conversation history, git state, and task lists — you lose time and mental state every time you switch.

Cortex is the missing command center: a Tauri-based desktop app that gives every project its own persistent brain, terminal environment, and AI chat — all resumable instantly.

Primary Goal
Zero context loss when switching between projects
Key Differentiator
Per-project AI brain with version-aware reference intelligence
OSS Strategy
Anthropic Claude for Open Source eligibility target

Code editor, multi-user collab, cloud sync, plugin marketplace

04 Core Features (MVP) MVP
Project Management
Add projects manually or auto-detect from folder path. Auto-detect git repo and tech stack type. Persist metadata including status (Active / Paused / Planning / Maintenance), last opened timestamp, and notes. Sidebar project list with search and recents.
Terminal Management
Spawn multiple terminal sessions per project using node-pty + xterm.js. Tab-based interface with rename, restart, kill, and clear. Terminal types: Shell, AI Session, Dev Server, Git. Terminals are project-bound and persist across sessions.
Per-Project AI Chat
Each project has a dedicated Claude conversation with stored history, context notes, and a custom system prompt. Chat interface with message history scroll, export, and clear. AI session data scoped per project — no context bleed across projects.
Project Notes & Tasks
Plain markdown notes with autosave per project. Task tracker with status: Pending / Doing / Done / Blocked. Simple, no-frills interface — productivity over aesthetics.
Git Integration
Live fetch of git status, current branch, uncommitted files, and last commit log. Git commands: status, branch, log, pull, push — executed via simple-git. Visual indicators for dirty state and branch name in project header.
Workspace Resume
On launch, Cortex restores the last active project: open terminals, active tab, AI chat position, and layout state. Zero setup on restart. This is the core productivity promise.

06 Intelligence Builder Module MVP

A structured learning layer so Cortex improves over time by remembering patterns, debugging knowledge, architecture decisions, and reusable solutions — per project.

Project Brain
Manual storage of project summary, architecture notes, known issues, and key decisions. Persisted in project_brain table. Injected into AI context automatically when chatting within a project.
Pattern Memory
Save reusable code patterns with title, description, tags, and source project. Marked as reusable or project-specific. Searchable across all projects.
Debug Memory
Store bug solutions: problem, root cause, solution, and source project. Prevents solving the same bug twice across projects. Reusable flag for cross-project applicability.

Note: MVP is manual-save only. Automatic pattern extraction via local models is post-MVP.

07 Reference Intelligence Module MVP

The killer feature. Maintain accurate, version-pinned reference data for APIs, SDKs, and CLI tools. AI retrieves only version-matching commands — eliminating the #1 source of hallucination in coding assistants: stale documentation.

Tool Registry
Add tools manually: name, category, doc URL, last updated. Track multiple versions per tool with release notes and dates.
Version-Aware Commands
Store commands per tool per version per OS (Linux / Windows / Mac). Mark deprecated commands with replacement. AI context only receives commands matching the selected version — never stale suggestions.
API Change Tracking
Log breaking changes per version: change type, old usage, new usage, notes. Prevents AI from suggesting removed APIs.
Future: Vector Search
Architecture is prepared for semantic search via ChromaDB or LanceDB + local embedding model. Not implemented in MVP — schema is forward-compatible.

— Development Constraints
—
No microservices, Docker, or Kubernetes. Monolithic architecture for MVP. Node.js sidecar + SQLite only.
—
No premature abstraction. Simple, readable TypeScript. Stable libraries only.
—
Performance targets: Launch under 3s · Memory under 300MB · 10 active projects · 5 simultaneous terminals.
—
Security: Local-only storage. No telemetry. No cloud transmission. Secure process spawning.
—
UI simplicity: Dark theme, minimal animations, VSCode-familiar layout. Productivity over aesthetics.
